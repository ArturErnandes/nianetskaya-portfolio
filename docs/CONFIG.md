# Project Configuration

Техническая спецификация конфигурации проекта.

## 1. Файл `data.json`

### Расположение

- Путь: корень репозитория, файл `data.json`.
- Загрузка выполняется при старте приложения в `backend/config.py`.
- Файл обязателен: при отсутствии или некорректной структуре backend не запустится.

### Пример

```json
{
  "app": {
    "host": "127.0.0.1",
    "port": 8080
  },
  "database": {
    "admin": "portfolio_user",
    "password": "example_db_password",
    "host": "127.0.0.1",
    "port": "5432",
    "db_name": "portfolio_db"
  },
  "admin": {
    "password_hash": "pbkdf2_sha256$200000$example_salt$example_hash"
  }
}
```

### Параметры

#### `app`

| Поле | Описание |
|------|----------|
| `host` | Хост, на котором слушает FastAPI/Uvicorn |
| `port` | Порт приложения |

#### `database`

| Поле | Описание |
|------|----------|
| `admin` | Пользователь PostgreSQL |
| `password` | Пароль PostgreSQL |
| `host` | Хост PostgreSQL |
| `port` | Порт PostgreSQL |
| `db_name` | Имя базы данных |

#### `admin`

| Поле | Описание |
|------|----------|
| `password_hash` | Хеш пароля администратора в формате `pbkdf2_sha256$iterations$salt$hash` |

## 2. Параметры окружения

В текущей реализации проект **не использует собственные переменные окружения** для конфигурации приложения.

- Источник настроек backend: только `data.json`.
- Параметры Nginx задаются в `nginx.conf`, не через env.

Если в будущем будет добавлена env-конфигурация, рекомендуется:

- хранить секреты в переменных окружения;
- не коммитить чувствительные данные в `data.json`;
- добавить карту переменных и приоритетов (env vs file) в этот документ.
